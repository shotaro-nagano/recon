# RECON — 卓球選手専用 AIコーチ

> 会話するほど・試合を読ませるほど精度が増す、**選手本人専用に育つAIコーチ**。
> 「CODENAME COACH 仕様書 v1.5（統合版）」の実装。製品名は **RECON**。

選手の承認済み試合データから **16タイプ（8コードネーム × α/Ω）** を実測し、強み・弱点・崩壊パターンが記録された「選手カルテ」が試合を重ねるほど育っていきます。PWA（プログレッシブ・ウェブ・アプリ）として実装されており、**1つのコードベースで PC / iOS / Android すべてに対応**します。

## 特長

- **育つコーチ** — 承認済みの試合が貯まるほどタイプ判定の精度（信頼区間）が上がり、UIが選手のタイプ色に染まっていく。
- **正直な数値主義** — すべての数値に `[サンプル数pt]`（必要なら `±信頼区間`）を併記。確定していない傾向は「可能性」と表現し断定しない。
- **会場で使える** — 戦術カードはオフラインで開ける（Service Worker プリキャッシュ）。
- **完全に端末内** — 試合データもカルテも端末のブラウザ内（IndexedDB）だけに保存。外部送信・共有・エクスポートは一切なし。
- **初めてでも迷わない** — 名前入力 → 診断 → 全画面の使い方ガイド（アニメ付き）→ かんたんモードで要点だけ表示。

## 主な機能

| 機能 | 概要 |
|---|---|
| **オンボーディング** | 初回起動で名前入力 → 7問の診断（約2分）で仮タイプを付与 → 全画面の使い方ガイド |
| **ホーム** | エンブレム＋コードネーム、精度メーター、4軸スコア、要確認、次のアクション |
| **コーチ（3モード）** | 試合前＝戦術カード / 試合後＝振り返り / 週次＝練習メニュー |
| **戦術カード** | 相手別の作戦をスクロールなし1画面・5項目で（会場でオフライン閲覧可・アニメ無効で最高可読） |
| **試合後レビュー** | 良かったデータを先出し → ほころび分析 → 「次の練習で1つだけ」提案 → カルテ更新承認 |
| **週次レポート** | 直近7日から優先順位つき練習メニュー、軸スコア推移、解消報告、β確認 |
| **試合データ管理** | CSV取り込み（貼り付け／ファイル）、**二層承認**（承認した試合だけが計算に使われる） |
| **カルテ** | 確定タグ／観察中／解消済み、崩壊ループ、サーブ別成績、コース別失点、練習履歴、コーチングメモ |
| **相手カルテ** | 暫定タイプ・3項目分析（サーブ傾向／レシーブの穴／勝負どころの癖）・対戦メモ |
| **AIチャット** | カルテ・タイプ・相性から答える相談チャット（後述） |
| **かんたんモード** | 既定ON。上級機能を隠して要点だけ表示。設定でフル機能に切替 |
| **設定** | ペルソナ（口調）3種、スキンA/B/C、シーズンオフ、データのスナップショット復元、デモ投入 |

### AIチャット（端末内・外部送信なし）

設定の「チャットのAI」で2つのエンジンから選べます。**どちらも端末内で動作し、外部サーバーには送信しません。**

- **かんたん応答（既定）** — 端末内の意図分類エンジン。あなたのカルテ・タイプ・相性・試合データから即答（無料・オフライン）。
- **端末のAI（実験的）** — ブラウザ内蔵AI（Chrome の Prompt API ＝端末上で動くモデル）で自由に会話。選手データ＋コーチ人格＋鉄則をモデルに渡して回答。**対応端末のみ**（非対応なら自動で「かんたん応答」にフォールバック）。

> いずれのエンジンも、フォームの細部・体調や怪我・深いメンタルは「監督・コーチに相談を」と人間へ振り、相性の助言には対戦サンプル数を必ず併記します。

## タイプシステム（16タイプ）

4つの軸（承認済み直近5試合・試合数ベースの指数減衰・95%信頼区間）から判定:

- **攻め/堅実（A/S）**, **勝負強さ/序盤型（C/N）**, **変化/王道（V/R）**, **先行/後半（F/L）**

8コードネーム × α（先行型）/Ω（後半に牙を剥く型）= 16タイプ:

| コードネーム | 読み | スタイル |
|---|---|---|
| BLADE | ブレイド | 王道正面の斬り込み |
| PHANTOM | ファントム | 変幻自在の攻撃 |
| BULLET | バレット | 一直線の超速攻 |
| JOKER | ジョーカー | 読めない変化トリック |
| FORTRESS | フォートレス | 何でも返す要塞 |
| SNIPER | スナイパー | 守りから一撃必中 |
| ANCHOR | アンカー | 絶対に切れない粘り |
| ORACLE | オラクル | 全部読んで先回り |

診断＝仮タイプ、承認済み5試合で実測タイプに進化。相性表は仕様書の生成ルールから構築し、実対戦データが貯まると自動補正します。

## プライバシー（最重要）

- 試合データ・カルテ・タイプ・弱点・崩壊ループを含む**全データは、この端末のブラウザ内（IndexedDB）にのみ保存**されます。
- **サーバーへの送信・共有・エクスポート機能はありません。**
- 「端末のAI」を選んだ場合もブラウザ内蔵モデルが端末上で動作し、外部送信はありません。
- このリポジトリのコードには個人データは含まれません（データは各利用者の端末内のみ）。

## 技術スタック

- **Vite + React 18 + TypeScript（strict）**
- **zustand**（状態管理）＋ **idb-keyval**（IndexedDB 永続化・スナップショット）
- **vite-plugin-pwa / Workbox**（オフライン・ホーム画面追加）
- フォント: Oswald（見出し）/ IBM Plex Mono（数値）/ Noto Sans JP（本文）
- ロジックは決定論的（v1ではLLM不要）。「端末のAI」選択時のみブラウザ内蔵AIを利用。

## スマホで使う / アプリとして入れる

本番ビルド（`dist/`）を HTTPS でホスティング（Vercel / Netlify / Cloudflare Pages など）した後:

- **iOS（Safari）**: サイトを開く → 共有 → 「ホーム画面に追加」
- **Android（Chrome）**: メニュー → 「アプリをインストール」
- **PC（Chrome/Edge）**: アドレスバーのインストールアイコンからアプリ化

インストール後は**オフラインでも全画面が開きます**。将来ストア配布が必要になれば [Capacitor](https://capacitorjs.com/) で `dist/` をそのままネイティブアプリにラップできます。

最初は **設定 →「デモデータを読み込む」**（架空選手・7試合分）で全画面の動きを確認できます。

## 試合CSVの形式

①得点チェックリストアプリ・②撮影判定アプリの出力を想定した「1試合1ファイル」形式:

```csv
# date: 2026-06-13
# opponent: 山田太郎
# tournament: 県リーグ
# result: 3-1
# source: 得点チェックリスト
set,my_score,opp_score,server,serve_type,serve_course,rally_length,third_ball_attack,winner
1,0,0,me,巻き込み,1,3,1,me
1,1,0,opp,ロング,4,5,0,opp
```

- `serve_course`: 1=バック前 2=ミドル前 3=フォア前 4=バック奥 5=ミドル奥 6=フォア奥
- `server` / `winner`: `me` または `opp`
- **取り込んだだけでは計算に使われません。承認した試合のみ**がカルテ・タイプ計算の対象です（絶対ルール）。

## プロジェクト構成

```
src/
├─ brand.ts             # アプリ名(RECON)など一元管理
├─ App.tsx              # ルーティング・ナビ・タイプ色浸透・使い方ガイド表示
├─ domain/              # ドメインロジック(UIに依存しない・テスト容易)
│  ├─ types.ts          #   全型定義
│  ├─ constants.ts      #   16タイプ定義・軸・コース・ペルソナ等
│  ├─ axisEngine.ts     #   4軸スコア(指数減衰・ジャックナイフCI)
│  ├─ typeEngine.ts     #   タイプ判定(境界帯・ヒステリシス・β校正)
│  ├─ compatibility.ts  #   相性表(生成ルール＋α/Ω補正＋データ自動補正)
│  ├─ insights.ts       #   サーブ別/コース別失点/確定タグ検出
│  ├─ pipeline.ts       #   二層承認パイプライン
│  ├─ persona.ts        #   口調(operator/passion/analyst)
│  ├─ accent.ts         #   タイプ色浸透システム
│  ├─ csv.ts            #   試合CSV入出力
│  ├─ demo.ts           #   デモデータ生成(シード固定)
│  └─ coach/            #   3モード＋チャットの生成ロジック
│     ├─ preMatch.ts    #     戦術カード
│     ├─ postMatch.ts   #     試合後分析
│     ├─ weekly.ts      #     週次メニュー
│     ├─ chat.ts        #     チャット意図分類＋端末AI用システムプロンプト
│     └─ deviceAI.ts    #     ブラウザ内蔵AI連携(世代差吸収・可否判定)
├─ store/useAppStore.ts # zustand + IndexedDB(スナップショット/ロールバック)
├─ components/          # Emblem(SVG)・PrecisionMeter・UI部品・TourOverlay
├─ screens/             # 各画面(ホーム/診断/コーチ/チャット/試合/カルテ/設定 等)
└─ styles/              # tokens.css(デザイントークン) / base.css

docs/CONTRACTS.md       # 画面実装の契約書(API・デザイン規則・画面仕様)
assets/imagegen_prompts.md  # ChatGPT用タイプ画像キャラデザプロンプト(8種)
public/                 # PWAアイコン・manifest 素材
scripts/                # アイコン生成・スクリーンショット補助
```
